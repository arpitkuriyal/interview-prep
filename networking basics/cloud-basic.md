# Cloud Computing

---

## 1. What is Cloud Computing?

**Answer:**
Cloud computing is the delivery of computing services (servers, storage, databases, networking, software) over the internet.

Simple:

> Instead of owning servers, you rent resources from cloud providers.

---

## 2. Types of Cloud (Deployment Models)

### Public Cloud:

* Shared resources
* Example: AWS, Azure

### Private Cloud:

* Dedicated to one organization

### Hybrid Cloud:

* Combination of public + private

**Answer:**

> Public is shared, private is dedicated, hybrid is a mix of both.

---

## 3. Service Models (VERY IMPORTANT)

### IaaS (Infrastructure as a Service):

* Provides virtual machines
* Example: EC2

### PaaS (Platform as a Service):

* Provides platform to build apps
* Example: Heroku

### SaaS (Software as a Service):

* Ready-to-use software
* Example: Gmail

**Answer:**

> IaaS gives infrastructure, PaaS gives platform, SaaS gives software.

---

## 4. What is Virtualization?

**Answer:**
Virtualization allows multiple virtual machines to run on a single physical machine.

Benefit:

> Better resource utilization.

---

## 5. What is a Virtual Machine (VM)?

**Answer:**
A VM is a software-based computer that runs an OS like a real machine.

---

## 6. What is a Container?

**Answer:**
A container packages an application with its dependencies.

Example:

* Docker

Difference from VM:

> Containers are lightweight and faster.

---

## 7. What is Scalability?

**Answer:**
Scalability is the ability to increase or decrease resources based on demand.

Types:

* Vertical (increase power)
* Horizontal (add more machines)

---

## 8. What is Load Balancing?

**Answer:**
Distributes traffic across multiple servers.

Benefit:

* High availability
* Better performance

---

## 9. What is High Availability?

**Answer:**
Ensures system is always up and running.

Achieved by:

* Multiple servers
* Failover systems

---

## 10. What is Fault Tolerance?

**Answer:**
System continues working even if some components fail.

---

## 11. What is Auto Scaling?

**Answer:**
Automatically increases or decreases resources based on traffic.

---

## 12. What is CDN (Content Delivery Network)?

**Answer:**
Stores content on multiple servers worldwide.

Benefit:

> Faster delivery by serving from nearest location.

---

## 13. What is Object Storage vs Block Storage?

### Object Storage:

* Stores data as objects
* Example: S3

### Block Storage:

* Stores data in blocks
* Example: EBS

**Answer:**

> Object storage is for files, block storage is for disks.

---

## 14. What is Serverless Computing?

**Answer:**
You run code without managing servers.

Example:

* AWS Lambda

---

## 15. What is Region & Availability Zone?

**Answer:**

* Region → Geographic location
* Availability Zone → Data center within region

Used for:

> High availability

---

## 16. What is Cloud Security?

**Answer:**
Protecting cloud systems and data.

Includes:

* Encryption
* Access control
* Network security

---

## 17. What is IAM (Identity and Access Management)?

**Answer:**
Controls who can access what resources.

---

## 18. What is Multi-Tenancy?

**Answer:**
Multiple users share the same cloud infrastructure securely.

---

## 19. What is CAP Theorem?

**Answer:**
A distributed system can provide only two of:

* Consistency
* Availability
* Partition Tolerance

Example:

> Systems choose trade-offs based on needs.

---

## 20. What is Event-Driven Architecture?

**Answer:**
System reacts to events (like user actions).

Example:

> Upload file → triggers processing

---

## 21. What is Infrastructure as Code (IaC)?

**Answer:**
Managing infrastructure using code.

Tools:

* Terraform
* CloudFormation

---

## 22. What is Disaster Recovery?

**Answer:**
Plan to recover systems after failure.

Includes:

* Backups
* Replication

---

## 23. What is Cost Optimization in Cloud?

**Answer:**
Reducing cloud expenses.

Techniques:

* Auto scaling
* Reserved instances
* Monitoring usage

---

## 24. What is Observability?

**Answer:**
Monitoring system performance.

Includes:

* Logs
* Metrics
* Traces
